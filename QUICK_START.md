# 快速启动指南

## 启动方式

### 方式一：普通启动（推荐）

双击运行: `start_with_ui.bat`

效果：
- 显示终端窗口
- 服务在前台运行
- 可查看实时日志
- 关闭窗口即停止服务

### 方式二：后台启动

双击运行: `start_background.vbs`

效果：
- 不显示终端窗口（后台静默运行）
- 服务在后台运行
- 需要手动访问 http://127.0.0.1:9131

### 方式三：手动停止

双击运行: `stop.bat`

## 访问地址

服务启动后，在浏览器中打开：

**http://127.0.0.1:9131**

## 使用步骤

1. **启动服务**
   - 双击 `start_with_ui.bat`
   - 等待显示 "Application startup complete"

2. **打开浏览器**
   - 访问 http://127.0.0.1:9131

3. **选择提供商**
   - 点击预设的提供商按钮（阿里、火山、Kimi、DeepSeek等）
   - 或点击"添加新提供商"

4. **填写配置**
   - 输入 Base URL
   - 输入 API Key
   - 输入模型ID（例如：deepseek-v3-2-251201）

5. **应用配置**
   - 点击"保存配置并应用"
   - 系统会自动重启 OpenClaw Gateway 服务

6. **切换模型**
   - 在底部的模型卡片中点击任意模型
   - 系统会立即切换到该模型并重启服务

## 服务管理

### 查看服务状态

在浏览器中访问：http://127.0.0.1:9131/api/config

返回JSON格式的配置信息

### 停止服务

- 方式1：关闭启动服务的终端窗口
- 方式2：双击运行 `stop.bat`
- 方式3：任务管理器中结束 python.exe 进程

### 重启服务

在 Web UI 中点击"重启服务"按钮

## 常见问题

### Q1: 双击 start.bat 后窗口闪退

A1: 这是正常的，因为 `start.bat` 已过时。请使用：
- `start_with_ui.bat`（显示窗口）
- `start_background.vbs`（后台运行）

### Q2: 访问 http://127.0.0.1:9130 显示"无法访问此网站"

A2: 可能原因：
- 服务还在启动中，请等待10-30秒
- 端口9131被其他程序占用
- 防火墙阻止了访问

**解决方法：**
1. 检查终端窗口是否显示 "Application startup complete"
2. 检查端口占用：`netstat -ano | findstr :9131`
3. 关闭防火墙或添加例外规则

### Q3: 切换模型后服务无法启动

A3: 检查配置是否正确：
- API Key 是否有效
- 模型ID 是否正确
- Base URL 是否可访问

查看后端日志中的错误信息

### Q4: 如何修改服务端口

A4: 编辑 `backend/main.py`，修改：
```python
uvicorn.run(app, host="127.0.0.1", port=9130, ...)
```

将 9131 改为其他端口（如 8080）
